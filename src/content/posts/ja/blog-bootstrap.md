## はじめに

私はこれまでにそこそこな量のコードを書いてきました。例えばゲームエンジンやそれを使ったゲーム、PC-9801用音源ドライバをWindowsで動かして自作ハードウェアを叩くソフトウェア等です。

しかしながら、それらのコードを書いた割には自分の利益があまりにも少なすぎます。

「じゃあブログをやって多少の広告収入をもらおう」となったのはいいものの、正直Webはバックエンド以外は本当に興味がありません。

なので、記事候補を自動で拾ってくる仕組みの設計は自分でちゃんと考えつつ、サイト本体はClaude Codeにまるっと作ってもらうことにしました。

## 記事候補の自動生成

まず、面白いネタを毎回自分で掘り起こすのは面倒なので、`blogscan` という自作ツールを作らせました。やっていることは単純で、4種類のソースから記事としても妥当なものを拾ってきてスコアリングしているだけです。

- 各リポジトリのgitコミット履歴（コミットクラスタ化して、試行錯誤の跡がある塊を高評価）
- Claude Codeとのセッション記録（コミットに残らない「捨てた案」を拾える）
- 自分用のメモリファイル（内省・意思決定の記録）
- 各リポジトリのREADME/docs（自作フォーマットやアルゴリズムの解説）

スコアリングは全部決定論的なルールベースで、LLMには一切投げていません。

```yaml
auto_promote:
  enabled: true
  confidence_threshold: 0.8
```

スコア(confidence)が一定(confidence_threshold)以上の候補は自動で下書き作成まで進みます。ただし進むのはあくまで下書き生成までで、実際にどれを記事にするかは自分で選ぶようにしました。ここを自動化するとAIが書いたマジで面白くない記事ばっかりになってしまうので、意図的に人間の判断を挟むようにしました。

## サイト本体

サイトは Astro で組んでもらいました。理由は、コード多めの記事とコンテンツコレクションの相性が良いこと、日英ルーティングが標準機能で賄えることらしいです。私は知りません。ホスティング方式やドメインといった判断が必要な部分だけこちらで決めて、実装は丸投げして進めました。

ホスティングは GitHub Pages を軸にしつつ、自宅のProxmox上のLXCコンテナで Cloudflare Tunnel（`cloudflared`）を動かす構成を採用しました。ポートを一切開けたくなかったのが理由で、`cloudflared` はコンテナからCloudflareへの outbound 接続だけで完結する。結果として構成は以下の通りで、かなりシンプルになったと思います。

```
nazet.jp（Cloudflare DNS） → GitHub Pages（直結）
api.nazet.jp（Cloudflare Tunnel） → 自前LXC → GitHub Pagesではどうにもならないような部分
```

## 現状とこれから

サイトはここにアクセスできているならわかる通り、 `https://nazet.jp/` で公開しました。とはいえ中身は About ページと空のBlogがあるだけで、実際の記事はまだほとんどありません。`blogscan` が溜めてくれた候補を見ながら、少しずつ記事にしていくつもりです。

## 余談

この記事はClaude Codeに下書きを書かせて、それを私が編集するという形で書いてみました。

書いてみて思ったこととして、Claude Codeの下書きはまあ面白くないです。「その記述は誰に需要があるの？」とか、「読者をLLMだと思っているのか？」というツッコミどころが当たり前のように出てきます。

つまらない記事を公開する趣味はないので、このワークフローは見直す必要がありそうです。
