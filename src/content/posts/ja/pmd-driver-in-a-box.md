> **これはサンプル記事です。** テンプレート（目次・シリーズナビ・シンタックスハイライト・タグ）の動作確認用のダミーコンテンツで、実際の記事はこれから執筆されます。

## 症状

1990年代のPC-98用音源ドライバのバイナリを、現代のRustプログラムから直接呼び出したい。しかしそのバイナリは8086向けの生の機械語であり、当時のOS/ハードウェア前提で書かれている。

## 誤った仮説

最初は命令セットを解析して丸ごと書き直す案を検討したが、命令数が多く現実的ではなかった。

## 根本原因

必要なのは「書き直し」ではなく「実行環境のエミュレーション」だった。

## 修正

`libx86emu` を組み込み、I/Oポート書き込みをトラップして仮想的なOPNAチップとして扱うことで、当時のバイナリをそのまま動かせるようにした。

```rust
fn handle_io_write(port: u16, value: u8) {
    // 仮想OPNAへのレジスタ書き込みとして扱う
    virtual_opna.write_register(port, value);
}
```

## 教訓

「動いているものをそのまま使う」ためのラッパーを書くという発想は、リバースエンジニアリングにおいて有効な選択肢になりうる。

次回は、この仮想OPNAの出力を実際のハードウェアに転送する部分について書く予定。
